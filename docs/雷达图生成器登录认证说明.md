## 1. 系统设计说明

### 1.1 为什么选择这样的设计

**前后端分离架构的需求：**
- 项目采用前后端分离架构（前端Vue3 + 后端SpringBoot）
- 前端和后端运行在不同的域名/端口
- 无法使用传统的Session/Cookie机制

**选择JWT认证的原因：**

| 传统Session | JWT |
|-------------|-----|
| 需要服务端存储Session信息 | 无状态，服务端无需存储 |
| 跨域问题复杂 | 天然支持跨域 |
| 占用服务器内存 | 减轻服务器负担 |
| 依赖Cookie | 可以存储在localStorage/sessionStorage |
| 水平扩展复杂 | 容易水平扩展 |

### 1.2 认证流程设计

```
┌─────────────┐
│  前端 Vue3 │
└─────┬───────┘
      │
      │ 1. 用户输入账号密码
      │
      ▼
┌─────────────────────────────────────────────────┐
│  2. POST /api/auth/login             │
│     Body: {account, password}          │
└─────────────┬───────────────────────────────┘
              │
              │ 3. 验证用户名/邮箱
              │ 4. BCrypt验证密码
              │
              ▼
     ┌────────────────────────┐
     │  用户存在且密码正确  │
     │  生成JWT Token     │
     └────────┬───────────┘
              │
              │ 5. 返回Token和用户信息
              │
              ▼
┌─────────────┐
│  前端存储Token  │
└─────┬───────┘
      │
      │ 6. 后续请求携带Token
      │
      ▼
┌─────────────────────────────────────────────────┐
│  7. POST /api/protected-resource        │
│     Header: Authorization: Bearer {token} │
└─────────────┬───────────────────────────────┘
              │
              │ 8. JwtAuthenticationFilter验证
              │ 9. 解析Token获取用户ID
              │
              ▼
         ┌───────────────────────┐
         │   允许访问受保护资源  │
         └───────────────────────┘
```

## 2. JWT技术讲解

### 2.1 什么是JWT

JWT（JSON Web Token）是一种开放标准（RFC 7519），用于在各方之间安全传输信息。

**JWT的结构（三部分）：**
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.
eyJzdWIiOiIxMjM0MjU3ODk4IiwibmFtZSI6ImFkbWluIn0.
SflKxwRJSMeKKF2QT4fwpCJ9yYXRfa2Q

└──────┬────────────────┬────────────┬─────────────┘
       │                │            │
    Header          Payload      Signature
  (头部信息)       (数据体)     (签名)
```

### 2.2 项目中的JWT配置

**配置位置：** `application.yml`
```yaml
jwt:
  secret: your-256-bit-secret-key-change-this-in-production  # 密钥，用于签名和验证
  expiration: 86400000  # 过期时间（毫秒），24小时
```

**配置类：** `JwtConfig.java`
```java
@Configuration
public class JwtConfig {
    @Value("${jwt.secret}")
    private String secret;  // 从配置读取密钥
    
    @Value("${jwt.expiration}")
    private Long expiration;  // 从配置读取过期时间
}
```

### 2.3 Token生成详解

**生成代码：** `JwtUtil.generateToken()`
```java
public String generateToken(Long userId, String username) {
    Date now = new Date();
    Date expiryDate = new Date(now.getTime() + jwtConfig.getExpiration());
    
    return Jwts.builder()
        .subject(String.valueOf(userId))           // 存储用户ID
        .claim("username", username)              // 存储用户名（自定义数据）
        .issuedAt(now)                        // 签发时间
        .expiration(expiryDate)                 // 过期时间
        .signWith(getSigningKey())              // 使用密钥签名
        .compact();                             // 生成JWT字符串
}
```

**生成的Token示例：**
```
eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.
eyJzdWIiOiIxIiwidXNlcm5hbWUiOiJhZG1pbiIsImlhdCI6MTY5NDkxMzI4fQ.
SflKxwRJSMeKKF2QT4fwpCJ9yYXRfa2Q
```

**解析后的Payload：**
```json
{
  "sub": "1",              // subject: 用户ID
  "username": "admin",    // 自定义数据：用户名
  "iat": 1694913318,     // issuedAt: 签发时间
  "exp": 1695000118      // expiration: 过期时间
}
```

### 2.4 Token验证详解

**验证代码：** `JwtUtil.validateToken()`
```java
public boolean validateToken(String token) {
    try {
        Jwts.parser()
            .verifyWith(getSigningKey())  // 使用相同的密钥验证签名
            .build()
            .parseSignedClaims(token);    // 解析Token
        return true;
    } catch (Exception e) {
        return false;  // 签名无效、过期、格式错误等
    }
}
```

**验证失败的情况：**
| 错误类型 | 原因 |
|-----------|------|
| SignatureException | Token被篡改，签名不匹配 |
| ExpiredJwtException | Token已过期 |
| MalformedJwtException | Token格式错误 |
| UnsupportedJwtException | 不支持的算法 |

### 2.5 密码安全设计

**为什么使用BCrypt？**
- 单向加密，无法反向解密（即使数据库泄露，也无法获取原始密码）
- 自带盐值，相同的密码每次加密结果不同
- 计算慢（故意设计），防止暴力破解

**加密代码：**
```java
BCryptPasswordEncoder passwordEncoder = new BCryptPasswordEncoder();

// 注册时加密
String encodedPassword = passwordEncoder.encode("123456");
// 结果: $2a$10$N9qo8uLOickgx2ZMRZoMyeIjZAcWT0Q3XzKfQwHm8Z

// 登录时验证
boolean matches = passwordEncoder.matches("123456", encodedPassword);
```

### 2.6 Spring Security集成

**SecurityConfig.java配置说明：**

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) {
        http
            // 1. 禁用CSRF（JWT不需要）
            .csrf(csrf -> csrf.disable())
            
            // 2. 配置CORS（允许前端跨域访问）
            .cors(cors -> cors.configurationSource(corsConfigurationSource()))
            
            // 3. 使用无状态Session（JWT不需要Session）
            .sessionManagement(session -> session.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            
            // 4. 配置访问控制
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()  // 登录注册接口无需认证
                .anyRequest().authenticated()                 // 其他接口需要认证
            )
            
            // 5. 添加JWT过滤器（在用户名密码过滤器之前执行）
            .addFilterBefore(new JwtAuthenticationFilter(jwtUtil), 
                         UsernamePasswordAuthenticationFilter.class);
        
        return http.build();
    }
}
```

**JwtAuthenticationFilter.java工作流程：**

```java
protected void doFilterInternal(HttpServletRequest request, 
                                    HttpServletResponse response,
                                    FilterChain filterChain) {
    // 1. 从请求头获取Token
    String token = getTokenFromRequest(request);  // 从Authorization: Bearer {token}提取
    
    // 2. 验证Token并设置认证信息
    if (StringUtils.hasText(token) && jwtUtil.validateToken(token)) {
        Long userId = jwtUtil.getUserIdFromToken(token);
        String username = jwtUtil.getUsernameFromToken(token);
        
        // 3. 创建认证对象并设置到Security上下文
        UsernamePasswordAuthenticationToken authentication =
            new UsernamePasswordAuthenticationToken(userId, null, Collections.emptyList());
        authentication.setDetails(new WebAuthenticationDetailsSource().buildDetails(request));
        SecurityContextHolder.getContext().setAuthentication(authentication);
    }
    
    // 4. 继续过滤器链
    filterChain.doFilter(request, response);
}
```

### 3.8 常见问题排查

**问题1：连接被拒绝（Connection Refused）**
- 检查后端是否启动：`ps aux | grep java`
- 检查端口是否被占用：`lsof -i :8080`
- 检查防火墙是否阻止

**问题2：数据库连接失败**
- 检查application.yml中的数据库配置（地址、用户名、密码、端口）
- 确保MariaDB已启动
- 测试数据库连接：`mariadb -u <用户名> -p -h <数据库地址>`（根据实际配置调整）

**问题3：JWT验证失败**
- 检查Token是否从Authorization头部正确提取（需"Bearer "前缀）
- 检查密钥是否与生成时一致
- 检查Token是否过期

**问题4：CORS错误**
- 前端端口与CORS配置是否匹配（3000或5173）
- 检查浏览器控制台的CORS错误
- 如果是本地开发，可以使用浏览器插件临时禁用CORS测试

## 4. 安全注意事项

### 4.1 生产环境部署前

**必须修改的配置：**
```yaml
# application.yml
jwt:
  secret: # 必须使用强随机密钥（至少256位）
  # 生成方法：openssl rand -base64 32
  expiration: 86400000  # 24小时（根据业务调整）
```

**密钥生成命令：**
```bash
# Linux/Mac
openssl rand -base64 64

# Windows（PowerShell）
[Convert]::ToBase64String((1..32 | ForEach-Object {[byte]::new}).value) -join '')
```

### 4.2 前端安全实践

**Token存储：**
```javascript
// ✅ 推荐：使用sessionStorage（关闭浏览器即清除）
sessionStorage.setItem('token', token);

// ⚠️ 可接受：使用localStorage（需设置过期时间）
localStorage.setItem('token', token);

// ❌ 不推荐：存储在URL中（会被记录到日志）
```

**Token使用：**
```javascript
// 在请求拦截器中统一添加Token
axios.interceptors.request.use(config => {
  const token = sessionStorage.getItem('token');
  if (token) {
    config.headers.Authorization = \`Bearer \${token}\`;
  }
  return config;
});

// 401自动跳转登录
axios.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      // Token过期或无效，跳转登录
      sessionStorage.removeItem('token');
      router.push('/login');
    }
    return Promise.reject(error);
  }
);
```

### 4.3 HTTPS部署

**开发环境：** HTTP可以接受

**生产环境：** 必须使用HTTPS
- 防止Token在传输中被窃取
- Spring Security配置：`http.requiresChannel().anyRequest().requiresSecure()`
